TECHNICAL ARCHITECTURE REPORT

1. WHAT YONO SHIELD ACTUALLY DOES

YONO Shield is a real-time security guardian that runs on the user's Android phone. The moment the app launches, it performs a chain of automated security checks. If any check fails, the user is blocked from proceeding — protecting them from fraud they may not even be aware of.

The Security Boot Sequence (runs every time the app opens or resumes):

1
Hardware Attestation (TEE) — Asks the phone's secure hardware chip: "Is this device genuine and unmodified?" Uses Google Play Integrity API. If the bootloader is unlocked or the OS is rooted → permanent lockout screen.
2
Accessibility Hijack Scan — Checks if any sideloaded app has Android Accessibility permissions (can read your screen, tap buttons, steal OTPs). If found → Red Alert with TTS audio warning.
3
Notification Snooper Scan — Checks if any sideloaded app has Notification Listener access (can read your OTPs, banking alerts). If found → Red Alert with TTS audio warning.
4
Omni-Channel Shield (background) — Continuously monitors ALL incoming notifications (SMS, WhatsApp, Telegram, Signal, etc.), extracts URLs, and scans them against the KAVACH backend in real-time.

2. THE FOUR PILLARS

The app has four main tabs, each addressing a different attack vector:

1
CRYPTOGRAPHIC GATEKEEPER (Radar Tab)

Attack it stops: Fake banking apps (brand impersonation clones)

How it works: The user taps "SCAN NOW". The app runs a 4-gate pipeline on the device:

  • Gate 1 — Root Detection: Scans for 8 known su binary paths, 5 root management packages, and test-keys in the Android build fingerprint.
  • Gate 2 — Trojan Scan: Iterates every installed app. Any sideloaded app with Accessibility or Overlay permissions is flagged as a trojan.
  • Gate 3 — Certificate Extraction: Uses Android PackageManager to extract the SHA-256 fingerprint of the target app's signing certificate.
  • Gate 4 — Hash Comparison: Compares the live hash against a database of known-good official hashes. Mismatch = brand impersonation.

Result: Green "VERIFIED" card if safe. Full-screen Red Alert with TTS audio if any gate fails.

2
OMNI-CHANNEL SHIELD (Shield Tab)

Attack it stops: Phishing links sent via any messaging app

How it works: Uses Android's NotificationListenerService to intercept notification text from every messaging app on the phone. Extracts URLs using regex, then POSTs each URL to the KAVACH Python backend for analysis.

The backend runs an 8-rule heuristic chain:

  • Suspicious TLD detection (.top, .xyz, .buzz, .tk)
  • Banking keyword + non-banking domain combination
  • Unicode homoglyph detection (е vs e, о vs o)
  • IP-based URL detection (http://192.168.x.x/login)
  • Shortened URL flagging (bit.ly, tinyurl)
  • Excessive subdomain chains (sbi.login.verify.evil.com)
  • Brand name in subdomain but wrong root domain
  • ML typosquatting detection — character-level TF-IDF cosine similarity against official SBI domains

Plus external threat intelligence: Google Safe Browsing API + VirusTotal (70+ AV engines).

3
THREAT SHIELD (Overlay Tab)

Attack it stops: User accidentally opening a malicious app

How it works: A native Android foreground service draws a full-screen TYPE_APPLICATION_OVERLAY window on top of malicious apps, physically blocking all user interaction until the threat is dismissed. Uses SYSTEM_ALERT_WINDOW permission.

4
SCAM SIMULATOR (Learn Tab)

Purpose: Gamified security education for rural, low-literacy users

How it works: A Tinder-style card swipe game presents 4 real-world scam scenarios (WhatsApp lottery, fake QR codes, OTP phone calls, fake Play Store apps). User swipes left for SCAM, right for SAFE. Text-to-Speech reads explanations aloud. Tracks a "Shield Score".

4. BACKEND PIPELINE DETAIL

When a WhatsApp message arrives:

1
Meta sends a webhook POST to /webhook with the message payload
2
KAVACH identifies the message type: text, image, or document
3
Text: Extract URLs → run 8-rule heuristic chain → query VirusTotal + Safe Browsing → return verdict
4
Image: Download from Meta CDN → decode QR codes (pyzbar) → extract URLs → run text pipeline
5
Document (APK): Verify magic bytes → extract signing certificates from ZIP → flag unsigned/tampered APKs
6
If phishing detected → reply warning to user via WhatsApp + trigger Automated Takedown Protocol

Automated Takedown Protocol (Background Task):

1
Phase 1 — OSINT: DNS resolution + WHOIS lookup (registrar, abuse contacts, nameservers)
2
Phase 2 — Google Report: Submit URL to Google Safe Browsing Lookup API v4
3
Phase 3 — SOC Ticket: Generate CERT-In/APWG-style abuse report → email to internal SOC analyst queue

6. KEY TECHNICAL DIFFERENTIATORS

  • Zero-Trust Architecture: Every security layer operates independently. A bypass at one layer does not compromise the others.
  • Omni-Channel Coverage: Unlike SMS-only solutions, KAVACH monitors ALL messaging apps via NotificationListenerService — WhatsApp, Telegram, Signal, Messenger, and any future app.
  • Hardware-Backed Attestation: Uses the device's Trusted Execution Environment (TEE) — a physically isolated security chip — making root detection impossible to spoof from software.
  • ML-Powered Typosquatting: Character-level TF-IDF detects sophisticated domain impersonation (e.g., "sttebankofindia.com" mimicking "statebankofindia.com").
  • Accessibility-First Design: Auto-speaking TTS warnings, massive icons, and high-contrast UI serve rural, low-literacy users who cannot read complex security alerts.
  • Automated Takedown: Not just detection — the system automatically gathers OSINT evidence and dispatches abuse reports to take down phishing domains.
  • Offline Resilience: SQLite store-and-forward ensures threat telemetry is never lost, even without network connectivity.
